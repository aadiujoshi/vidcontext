# vidcontext

`vidcontext` is a powerful, self-contained node module that allows you to easily extract frames from any video at a given time interval, filter out duplicate or mostly identical frames using Perceptual Hashing, and then automatically describe each unique frame in extreme detail using the **Gemini 1.5 Flash Vision API**. 

It handles bundling `ffmpeg` directly (via `ffmpeg-static`) out of the box, meaning it works immediately with zero external dependencies to install locally!

## Installation

```bash
npm install vidcontext
```

## Basic Usage

```typescript
import { processVideo } from 'vidcontext';

async function main() {
  const results = await processVideo('./path/to/your/video.mp4', {
    interval: 5, // Extract a frame every 5 seconds
    apiKey: 'YOUR_GEMINI_API_KEY', // Required: Google Gemini API Key
    verbose: true // Outputs debugging info in the console
  });

  console.log(results);
}

main();
```

### Result Format
The function returns a Promise wrapping an array of `VidContextResult` objects:
```typescript
[
  {
    timestamp: 0, // Approximate timestamp in seconds
    hash: '8f000b0f4f030a0e', // Computed Perceptual Hash
    description: 'A detailed description of the frame generated by Gemini...',
    imagePath: '/tmp/vidcontext-xxx/frame_0001.jpg' // (Only present if retainFrames is true)
  },
  // ...
]
```

## API Options

The `processVideo` method accepts an `options` object with the following properties:

| Option | Type | Default | Description |
|---|---|---|---|
| `interval` | `number` | **Required** | The time interval in seconds between frame extractions. |
| `apiKey` | `string` | **Required** | Your Google Gemini API Key. |
| `model` | `string` | `'gemini-1.5-flash'` | The Gemini model to use for generating text. |
| `prompt` | `string` | `"Describe this frame in extreme detail."` | The prompt sent to Gemini with the image. |
| `hashThreshold` | `number` | `10` | The tolerance (Hamming Distance) between perceptual hashes. A lower number is stricter (requires images to be exactly identical to count as duplicates). A higher number will allow more variance before considering a frame unique. |
| `retainFrames` | `boolean` | `false` | If set to `true`, the OS temporary folder holding the extracted images will **not** be deleted after processing completes. |
| `verbose` | `boolean` | `false` | If `true`, logs debug information regarding frame extraction, hash distances, and API execution times to the console. |
